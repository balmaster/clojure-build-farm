развертывание = что? + куда? + как?

система = инсталляционный пакет + схема развертывания + параметры среды

развертывание выполняется с компьютера, который называется сервером инсталляции
при развертывании, для взаимодействия с удаленными серверами используются стандартные 
средства коммуникации и аутентификации: unix (ssh,scp), windows (winrm,netbios)  

инсталляционный пакет формируется из компонентов, находящихся в хранилище на сервере
инсталляции. компоненты поставляются в составе дистрибутивов и патчей к отдельным 
дистрибутивам 

все компоненнты из всех дистрибутивов находятся в общем хранилище
каждый компонент имеет уникальное имя и версию

если в двух дистрибутивах есть компоненты с совпадающими именем и версией то они считаются
идентичными

компонентом может быть отдельное приложение, j2ee приложение развертываемое на сервере
приложений, набор файлов, скрипты формирования/миграции бд, и тд . система не накладывает 
ограничений на типы компонентов и ее функциональность может быть расширена путем реализации
адаптеров для новых типов компонентов

адаптер определяет, как выполнять отдельные задачи: установить приложение, обновить приложени
удалить приложение, запустить приложение, остановить приложение,рестартовать приложение, 
запустить сервер приложений, рестартовать сервер приложений, остановить сервер приложений
также агент определяет способ снятия статистики показателей работы с сервера приложений

отдельное приложение, набор конфигов
конфигурировать нужно приложение под сервер
